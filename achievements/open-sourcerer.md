# 🌊 Open Sourcerer

> Have pull requests merged in multiple public repositories in the same month.

**Difficulty:** 🟡 Medium  
**Tiers:** 1 (no tiers)  
**Time to earn:** Depends on open source contributions

---

## 📋 How to Earn It

You need PRs merged in **at least 2 different public repos** within the **same calendar month**.

### Step 1 — Find a beginner friendly issue
Go to GitHub and search:
```
is:issue is:open label:"good first issue" language:Python
```
Or visit: **goodfirstissue.dev**

### Step 2 — Fork the repo
- Go to the repo → click **"Fork"** (top right)
- This creates a copy in your account

### Step 3 — Make your fix
- Find the file that needs changing
- Click the **pencil ✏️ icon** to edit
- Make your fix (typo, documentation, small bug)

### Step 4 — Commit and open a PR
- Scroll down to **"Commit changes"**
- Select **"Create a new branch"**
- Click **"Propose changes"**
- Click **"Create pull request"**
- In the description write `Closes #ISSUE_NUMBER`

### Step 5 — Wait for merge
- The maintainer will review and merge your PR
- They may ask for small changes first

### Step 6 — Repeat in a DIFFERENT repo
Do the same process in a second repo **within the same month**!

## ✅ Tips

- Fix **documentation or typos** first — easiest to get approved
- Always **comment on the issue** before starting: *"I'd like to work on this!"*
- Keep PRs **small and focused** — one fix per PR
- Be **responsive** to maintainer feedback

---

## ⚠️ Common Mistakes

- Doing PRs in the **same repo** twice — must be different repos
- PRs in **different months** — must be the same calendar month
- Opening PRs but **not getting them merged**
